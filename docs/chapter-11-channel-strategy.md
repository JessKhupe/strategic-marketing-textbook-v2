# Chapter 11: Channel Strategy & Omnichannel Orchestration

!!! note "Bloom's Level & Duration"
    **Bloom's Level:** Analyze, Evaluate  
    **Duration:** 2 weeks

## Summary

Students move beyond multichannel presence to design omnichannel systems that deliver seamless experiences across touchpoints. The emphasis is on coordination, timing, and measurement.

## Learning Objectives

1. Design omnichannel orchestration architectures coordinating 6+ channels.  
2. Build channel‑specific AI optimization strategies for paid, owned, and earned media.  
3. Create unified customer profiles enabling cross‑channel experiences.  
4. Measure omnichannel performance with attribution spanning online and offline.

## Core Theory

- Multichannel vs omnichannel distinctions
- Paid / owned / earned dynamics
- Cross‑channel journey mapping and unified profiles
- Real‑time channel switching and handoffs

## Frameworks

| Framework | Purpose |
|---|---|
| Omnichannel Orchestration Map | Visualize channel roles and handoff logic |
| Channel Role Framework | Define channel responsibilities by funnel stage |
| Cross‑Channel Attribution Model | Guide measurement across touchpoints |

## AI Tool Applications

| Tool | Use case |
|---|---|
| Sprinklr / Adobe | Enterprise orchestration and content management |
| Salesforce / Iterable | Unified customer profiles and orchestration |
| Klaviyo / Attentive | Channel-specific personalization (email, SMS)

## Ethical Checkpoints

!!! warning "Ethical Checkpoints"
    - **Channel fatigue:** Limit frequency and coordinate across channels to avoid overwhelm.  
    - **Location data ethics:** Use precise location data responsibly.  
    - **Unified profiles & privacy:** Ensure consent covers cross‑channel stitching.
